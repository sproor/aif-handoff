# Prettier drift in codex model discovery retry test

**Date:** 2026-04-08 17:39
**Files:** packages/runtime/src/__tests__/codexModelDiscovery.test.ts
**Severity:** low

## Problem

The PR `#36` lint workflow failed in the `format:check` step. Prettier reported
`packages/runtime/src/__tests__/codexModelDiscovery.test.ts` as not matching the
repository formatting rules, so GitHub Actions exited with code 1 before ESLint
could continue.

## Root Cause

The previous fix for the CI retry-warning assertions updated the test logic but
the resulting line wrapping was not normalized with Prettier before pushing.
Because the repository enforces formatting in CI, a semantically correct change
still failed the lint workflow.

## Solution

Ran Prettier on `packages/runtime/src/__tests__/codexModelDiscovery.test.ts` and
verified both the targeted file and the repository-wide `format:check` command
pass. No runtime behavior changed; only formatting was updated.

## Prevention

- After touching tests or long string comparisons, run `npm run format:check`
  locally before pushing.
- Treat formatting checks as required validation for CI-only fixes, not as an
  optional cleanup step.
- Keep small follow-up PR fixes minimal and always verify the exact workflow
  step that failed.
